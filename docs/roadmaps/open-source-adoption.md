# Open Source Adoption Roadmap

## Goal

Turn the repository from "interesting prototype" into "credible open-source tool people can actually adopt".

## Scope

- README quality
- contributor onboarding
- release discipline
- maturity messaging
- project status visibility

## Current state

- repo is public
- CI is green
- README exists in English and Spanish
- contributors, credits, and release baseline are in place

## Milestones

### Milestone 1 — Strong onboarding

- keep quickstart short
- keep install/troubleshooting honest
- keep credits and scope explicit

### Milestone 2 — Cleaner contribution workflow

- clear issue/PR templates if needed
- stronger changelog discipline
- roadmap traceability by area

### Milestone 3 — Beta readiness

- real feedback from external users
- fewer environment-specific assumptions
- clearer support expectations

## Done means

- an outsider can understand the project, trust its current maturity level, and contribute without guessing

## Non-goals

- pretending the project is mature before it earns that status
